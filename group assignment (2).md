### 🎯 Core Application Requirements
Your team can build any web application, provided it strictly meets these 4 fundamental requirements:
1.  **Architecture:** Must use a Client-Server architecture.
2.  **Authentication:** Users must be able to Register, Login, and Logout.
3.  **Data Persistence:** User data must be saved in the database and persist between sessions.
4.  **Social/Interactive Element:** Users **must** be able to view data generated by other users (e.g., leaderboards, public posts, shared schedules, etc.).

### 💻 Technical Specifications (The Stack)
You must strictly follow the allowed technology stack. Using forbidden technologies will result in penalties.
*   **Core Tech (Allowed & Required):** 
    *   HTML, CSS, JavaScript, jQuery.
    *   **CSS Framework (Choose ONLY one):** Bootstrap, Tailwind, SemanticUI, *or* Foundation.
    *   **Backend:** Flask + plugins taught in lectures.
    *   **Database:** SQLite via SQLAlchemy (ORM).
    *   **Communication:** AJAX and/or Websockets.
*   **Forbidden Tech:** Frontend frameworks (React, Angular, Vue, etc.), other databases (MySQL, PostgreSQL), and advanced CSS preprocessors (like directly using SASS).
*   **Third-Party Libraries (Allowed):** You can freely use specific JS/Python libraries for extra features (e.g., Chart.js for graphs, OpenAI API, Leaflet for maps, NLP libraries, Font Awesome).

### 📊 Marking Rubric (How you get graded)

#### 1. Overall Application (Group Marks - shared equally)
*   **Content & Design:** All requirements met; visually appealing, intuitive UI/UX, and provides actual value to users.
*   **HTML/CSS:** Valid, responsive design (adapts to screen size), utilizing Jinja templates properly.
*   **JavaScript:** Clean code, includes DOM manipulation, form validation, and **AJAX** following best practices.
*   **Flask & Database:** Well-organized routing, non-trivial data manipulation. Good DB schema design with maintainable models (evidence of DB migrations expected).
*   **Testing (CRITICAL):** Must have **5+ Unit Tests** AND **5+ Selenium Tests**. Selenium tests must run against a live local server.
*   **Security (CRITICAL):**
    *   Passwords must be stored as **salted hashes**.
    *   Forms must use **CSRF tokens** to prevent attacks.
    *   Environment variables (secrets) must be securely stored in configuration files (not hardcoded).

#### 2. Agile Development Process (Individual Marks)
*This is based on your personal Git/GitHub activity. Ensure your local Git username/email matches your GitHub account!*
*   **Engagement:** Active participation in Checkpoint meetings.
*   **Commits:** Frequent, regular commits with descriptive, meaningful commit messages.
*   **Issues:** Active use of GitHub Issues to track both bugs (with reproduction steps) and missing features.
*   **Pull Requests (PRs):** Meaningfully named PRs used to merge new features and fix bugs.
*   **Teamwork / Code Review:** Evidence of collaboration—discussing issues, reviewing teammates' PRs, and addressing feedback before merging.

#### 3. Bonus / Teamwork Marks
*   A small number of bonus marks can be allocated internally by the group to reward members who went "above and beyond".
*   *Warning:* If a member fails to make meaningful contributions, their individual final mark will be reduced by the unit coordinators.
